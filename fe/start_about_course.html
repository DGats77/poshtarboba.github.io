Цільова аудиторія курсу

Цілеспрямовані молоді люди, що прагнуть працювати у сфері ІТ.

Поріг входження: володіння комп'ютером на хорошому рівні (вміння самостійно встановлювати програми, вирішувати проблеми зі своїм комп'ютером), посереднє знання англійської мови, вміння користуватися пошуком в інтернеті.

На кого не розраховано курс:

- на зайнятих людей, що не можуть приділяти достатньо часу навчанню, виконанню домашніх завдань та проектів;
- на тих, хто приходить лише слухати лекції, не готовий задавати питання;
- на людей, які з комп'ютером "на Ви".

Етапи створення сайту

Сучасні сайти створюють цілі команди професіоналів різних галузей ІТ-технологій. Сьогодні існує дуже багато різних технологій, одна людина просто не в змозі охопити повністю всі знання та технології для створення сайту з нуля до самого кінця.

Створення сайту поділяється на такі основні етапи:

- Керування проектом - цим займається менеджер проекту, комунікабельна людина, що об’єднує команду і контролює виконання проекту на всіх ланках.

- Дизайн, інтерфейс сайту - створення дизайн-макетів сторінок сайту, шаблонів, цим займаються веб-дизайнери та UI/UX-інженери.

- Фронтенд (frontend) - сукупність дій, в результаті яких макети сайта перетворюються на веб-сторінки:

- порізка - вирізання з макету всіх необхідних декоративних елементів, фотографій і т.п.;

- верстка - створення моделі даних майбутніх веб-сторінок в форматі HTML;

- стилізація - візуальне оформлення моделі даних, щоб вона набула зовнішнього вигляду відповідно до дизайн-макетів;

- скриптування - написання скриптів, підключення бібліотек для оживлення веб-сторінок анімацією, динамічністю.

- Бекенд (backend) - перенесення готових веб-сторінок на серверну частину, створення шаблонів для CMS:

- бази данних - створення та налаштування необхідних баз даних для роботи проекту;

- CMS - пристосування веб-сторінок для системи керування контентом, створення шаблону;

- хостінг - пошук, реєстрація, купівля хостингу, що відповідає вимогам проекту, реєстрація доменних імен;

- тестування - випробування проекту на навантаження, пошук помилок, недоліків, вразливостей.

- Впровадження, розкрутка (SEO), реклама.

- Обслуговування, наповнення контентом.

Процес навчання

Викладач стає вашим ментором на найближчі рік-два, поки ви не досягнете такого рівня, що не потребуватимете постійних консультацій.

Завжди можете звертатися з питаннями до викладача по вайберу, скайпу, в соц-мережах чи приходити на консультації, і якщо викладач має змогу - з радістю допоможе.

Перед кожною лекцією та після лекції студенти можуть отримати консультацію у викладача по пройденому матеріалу, домашнім завданням, власним проектам.

На лекції розглядається теорія та виконуються прості практичні завдання.

В кінці заняття є контрольні питання та домашні завдання.

Домашні завдання є обов'язковими для засвоєння матеріалу.

Деякі теми вам будуть даватися на самоопрацювання.

Загалом для успішного завершення курсу студент щодня повинен виділяти не менше 3-5 годин на самостійну роботу:
- перечитування лекційного матеріалу;
- виконання домашніх завдань;
- самостійне опрацювання матеріалів на сторонніх ресурсах;
- проходження сторонніх курсів, наприклад, codeCademy;
- читання статей по фронтенду та ІТ-технологіям на сторонніх ресурсах, наприклад, habrahabr, frontender, тематичних групах в соц-мережах;
- дослідження ринку розробки на сайтах фріланса;
- розробка своїх власних проектів.

Якщо ви не маєте змоги виділяти щодня стільки часу через сімейні обставини, навчання, роботу, природню лінь - цей курс може виявитися для вас марною тратою грошей та часу.

Після вивчення HTML та CSS якщо ви будете впевнені в своїх силах та матимете час - можете зробити декілька тестових версток і менеджери студії почнуть давати вам реальні оплачувані проекти.

Фронтенд - основа знань

Всі спеціалісти веб-девелопмента (створення сайтів) повинні знати фронтенд-частину. Комусь це потрібно в більшій мірі, комусь вистачить основ.

Дизайн - кожен дизайнер повинен могти зверстати свої макети хоча-б на базовому рівні для того, щоб розуміти принципи формування елементів веб-сторінки.

Бекенд - спеціаліст по бекенду повинен перетворювати HTML-сторінки на шаблони для CMS, вносити деякі правки, фікси, тому він повинен знати HTML, CSS, JS на рівні не менше середнього.

SEO - спеціаліст по розкрутці сайтів повинен адаптувати веб-сторінки до вимог пошукових систем, він повинен досконало знати HTML, розуміти семантику HTML5, розбиратися у стилях.

Розробник мобільних додатків - інтерфейс додатків створюється або аналогічно веб-сторінкам, або на XML і має багато спільних рис з фронтенд-розробкою сайтів.

Фронтенд - спеціаліст по верстці, мусить знати всі тонкощі HTML, CSS, JS.

Навички фронтенд-розробника для здобуття хорошої роботи

Робота з графічним редактором Adobe Photoshop, Adobe Illustrator, Sketch, Avocode.

Дані: HTML, XHTML, XML, JSON, бази даних.

Стилі: CSS, LESS/SASS, зборщики.

Різноманітні фреймворки, CMS.

Скрипти: JavaScript, jQuery, Angular, React, CoffeeScript, TypeScript...

Основи php, структура проекту, темплейтів.
